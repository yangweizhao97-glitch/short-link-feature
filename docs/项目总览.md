# 短链接项目总览

## 当前状态

当前目录包含短链接功能的规格、API 设计、数据库设计、代码实现、验证记录、Review 记录和工作记录。

该项目目前是一个 TypeScript 原型实现，不是完整 Web 服务。当前实现了框架无关的处理函数：

1. `POST /api/short-links` 的创建逻辑。
2. `GET /:code` 的跳转逻辑。
3. JSON 文件仓储作为本地开发替身。

## 已知问题

当前 Review 已记录以下重点问题：

1. 已禁用短链接仍会跳转。
2. 访问计数不是原子更新。
3. 缺少创建接口和跳转接口频率限制。
4. 短码格式过宽，允许任意长度。
5. URL 校验无法拦截解析到内网 IP 的域名。
6. 跳转使用 `originalUrl`，没有使用 `normalizedUrl`。
7. TypeScript 使用 dev 版本，建议改为稳定版本。

详细内容见：

1. `docs/安全Review.md`
2. `docs/当前DiffReview.md`

## 下一步建议

建议按以下顺序推进：

1. 修复 `GET /:code` 对 `status !== "active"` 的处理。
2. 限制短码长度为 6 到 8 位。
3. 给创建和跳转接口设计并实现限流。
4. 把跳转目标从 `originalUrl` 调整为 `normalizedUrl`。
5. 将 JSON 仓储替换为真实数据库仓储，访问计数使用原子更新。
6. 补充结构化日志实现，遵守 `docs/结构化日志设计.md`。
7. 在网络可用时重新执行 `npm audit --audit-level=high`。
