# 短链接服务结构化日志设计

## 目标

设计短链接服务的结构化日志字段，方便排查：

1. 创建短链接失败。
2. 短链接跳转失败。
3. 访问统计异常。

日志设计原则：

1. 记录事件，不记录散文。
2. 每条日志必须有稳定的 `event` 名称。
3. 每条日志必须带 `requestId`，方便串联一次请求。
4. 不记录完整 URL query。
5. 不记录 Cookie、Authorization、Token、完整请求头。
6. URL 相关日志只记录 domain、scheme、pathHash 等低敏字段。
7. 用户 IP 如需记录，只记录 `ipHash`，不记录明文 IP。

## 日志级别约定

| level | 使用场景 | 示例 |
|---|---|---|
| `info` | 正常关键业务事件 | 短链接创建成功、跳转成功 |
| `warn` | 可预期失败或被拦截 | URL 校验失败、短码不存在、短链过期 |
| `error` | 系统异常或数据不一致 | 数据库写入失败、访问计数更新失败 |
| `debug` | 本地诊断细节 | 生产默认关闭 |

## 通用字段

所有日志事件都建议包含以下字段。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `event` | string | 是 | 稳定事件名 |
| `level` | string | 是 | `info` / `warn` / `error` / `debug` |
| `requestId` | string | 是 | 请求 ID |
| `timestamp` | string | 是 | ISO 8601 时间 |
| `service` | string | 是 | 服务名，例如 `short-link-service` |
| `environment` | string | 是 | 环境，例如 `dev` / `staging` / `prod` |
| `route` | string | 是 | 路由模板，例如 `/api/short-links`、`/:code` |
| `method` | string | 是 | HTTP 方法 |
| `statusCode` | number | 否 | HTTP 响应状态码 |
| `durationMs` | number | 否 | 请求耗时 |
| `clientIpHash` | string | 否 | 客户端 IP 哈希 |
| `userId` | string | 否 | 登录用户 ID；匿名访问不填 |

禁止字段：

1. `authorization`
2. `cookie`
3. `token`
4. `password`
5. `rawHeaders`
6. `fullUrl`
7. `fullQuery`
8. 明文 IP

## URL 脱敏字段

短链接服务会处理用户提交的长链接。日志中不要记录完整 URL，尤其不要记录 query。

推荐字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `targetScheme` | string | `http` / `https` |
| `targetDomain` | string | 目标域名，例如 `example.com` |
| `targetPort` | number / null | 端口；默认端口可省略 |
| `targetPathHash` | string | path 的哈希，排查同一路径问题时使用 |
| `targetQueryPresent` | boolean | 是否存在 query |
| `targetFragmentPresent` | boolean | 是否存在 fragment |
| `targetUrlHash` | string | 归一化 URL 的哈希，用于关联同一 URL |

不推荐字段：

| 字段 | 原因 |
|---|---|
| `originalUrl` | 可能包含敏感 query |
| `normalizedUrl` | 可能包含敏感 query |
| `query` | 可能包含 token、邮箱、手机号、订单号 |
| `referer` 原文 | 可能包含敏感 query |

## 事件一：短链接创建成功

事件名：

```text
short_link_create_succeeded
```

level：

```text
info
```

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `shortLinkId` | string | 是 | 短链接 ID |
| `shortCode` | string | 是 | 生成的短码 |
| `shortCodeLength` | number | 是 | 短码长度 |
| `targetScheme` | string | 是 | 目标 URL 协议 |
| `targetDomain` | string | 是 | 目标域名 |
| `targetPathHash` | string | 否 | path 哈希 |
| `targetQueryPresent` | boolean | 是 | 是否包含 query |
| `expiresAt` | string / null | 是 | 过期时间 |
| `isDuplicate` | boolean | 否 | 是否复用已有短链 |

示例：

```json
{
  "event": "short_link_create_succeeded",
  "level": "info",
  "requestId": "req_123",
  "timestamp": "2026-06-14T12:00:00.000Z",
  "service": "short-link-service",
  "environment": "prod",
  "route": "/api/short-links",
  "method": "POST",
  "statusCode": 201,
  "durationMs": 18,
  "clientIpHash": "ip_hash_abc",
  "userId": "user_123",
  "shortLinkId": "sl_123",
  "shortCode": "abc123",
  "shortCodeLength": 6,
  "targetScheme": "https",
  "targetDomain": "example.com",
  "targetPathHash": "path_hash_abc",
  "targetQueryPresent": true,
  "expiresAt": null,
  "isDuplicate": false
}
```

## 事件二：短链接创建失败

事件名：

```text
short_link_create_failed
```

level：

1. URL 校验失败：`warn`
2. 系统错误：`error`

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `errorCode` | string | 是 | 业务错误码 |
| `failureReason` | string | 是 | 稳定失败原因 |
| `targetScheme` | string | 否 | 能解析时记录 |
| `targetDomain` | string | 否 | 能解析时记录 |
| `targetQueryPresent` | boolean | 否 | 能解析时记录 |
| `urlLength` | number | 否 | 原始 URL 长度 |
| `shortCodeAttemptCount` | number | 否 | 生成短码尝试次数 |

`failureReason` 建议值：

| failureReason | 说明 |
|---|---|
| `url_required` | URL 缺失 |
| `invalid_url` | URL 格式非法 |
| `unsupported_scheme` | 非 `http` / `https` 协议 |
| `url_not_allowed` | localhost、内网 IP 或禁止域名 |
| `short_code_generation_failed` | 短码生成失败 |
| `repository_error` | 存储层错误 |

示例：

```json
{
  "event": "short_link_create_failed",
  "level": "warn",
  "requestId": "req_124",
  "timestamp": "2026-06-14T12:01:00.000Z",
  "service": "short-link-service",
  "environment": "prod",
  "route": "/api/short-links",
  "method": "POST",
  "statusCode": 400,
  "durationMs": 4,
  "clientIpHash": "ip_hash_def",
  "userId": "user_123",
  "errorCode": "UNSUPPORTED_URL_SCHEME",
  "failureReason": "unsupported_scheme",
  "targetScheme": "javascript",
  "urlLength": 19
}
```

## 事件四：短链接跳转失败

事件名：

```text
short_link_redirect_failed
```

level：

1. 短码不存在、过期、禁用：`warn`
2. 存储错误、计数错误：`error`

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `shortCode` | string | 否 | 请求中的短码；格式非法时可不填或脱敏 |
| `shortCodeLength` | number | 否 | 短码长度 |
| `errorCode` | string | 是 | 错误码 |
| `failureReason` | string | 是 | 稳定失败原因 |
| `shortLinkId` | string | 否 | 找到记录时记录 |
| `expiresAt` | string / null | 否 | 找到记录时记录 |
| `linkStatus` | string | 否 | `active` / `disabled` |

`failureReason` 建议值：

| failureReason | 说明 |
|---|---|
| `invalid_short_code` | 短码格式非法 |
| `short_code_not_found` | 短码不存在 |
| `short_link_expired` | 短链接已过期 |
| `short_link_disabled` | 短链接已禁用 |
| `repository_error` | 存储层错误 |

示例：

```json
{
  "event": "short_link_redirect_failed",
  "level": "warn",
  "requestId": "req_126",
  "timestamp": "2026-06-14T12:03:00.000Z",
  "service": "short-link-service",
  "environment": "prod",
  "route": "/:code",
  "method": "GET",
  "statusCode": 404,
  "durationMs": 3,
  "clientIpHash": "ip_hash_jkl",
  "shortCode": "NOPE12",
  "shortCodeLength": 6,
  "errorCode": "SHORT_LINK_NOT_FOUND",
  "failureReason": "short_code_not_found"
}
```

## 事件六：访问统计更新失败

事件名：

```text
short_link_visit_count_update_failed
```

level：

```text
error
```

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `shortLinkId` | string | 否 | 短链接 ID |
| `shortCode` | string | 是 | 短码 |
| `errorCode` | string | 是 | 稳定错误码 |
| `failureReason` | string | 是 | 失败原因 |
| `updateMode` | string | 是 | `atomic` / `read_modify_write` |
| `retryable` | boolean | 是 | 是否可重试 |

`failureReason` 建议值：

| failureReason | 说明 |
|---|---|
| `record_not_found_during_update` | 查询成功后更新时记录不存在 |
| `repository_write_failed` | 写入失败 |
| `concurrent_update_conflict` | 并发更新冲突 |
| `invalid_visit_count` | 访问数异常 |

示例：

```json
{
  "event": "short_link_visit_count_update_failed",
  "level": "error",
  "requestId": "req_128",
  "timestamp": "2026-06-14T12:05:00.000Z",
  "service": "short-link-service",
  "environment": "prod",
  "shortLinkId": "sl_123",
  "shortCode": "abc123",
  "errorCode": "VISIT_COUNT_UPDATE_FAILED",
  "failureReason": "repository_write_failed",
  "updateMode": "read_modify_write",
  "retryable": true
}
```

## 推荐查询方式

### 排查创建失败

按 `event` 和 `errorCode` 聚合：

```text
event = short_link_create_failed
group by errorCode, failureReason
```

重点关注：

1. `UNSUPPORTED_URL_SCHEME` 是否异常升高。
2. `URL_NOT_ALLOWED` 是否集中在某些 `targetDomain`。
3. `SHORT_CODE_GENERATION_FAILED` 是否出现。

### 排查跳转失败

按 `event` 和 `failureReason` 查询：

```text
event = short_link_redirect_failed
group by failureReason
```

重点关注：

1. `short_code_not_found` 是否异常升高，可能表示短码枚举。
2. `short_link_expired` 是否集中出现，可能是过期策略影响用户。
3. `short_link_disabled` 是否对应安全封禁。

### 排查访问统计问题

查询访问统计失败：

```text
event = short_link_visit_count_update_failed
```

重点关注：

1. `repository_write_failed`
2. `concurrent_update_conflict`
3. `visitCountAfter` 是否小于 `visitCountBefore`

## 安全脱敏规则

必须遵守：

1. 不记录完整原始 URL。
2. 不记录完整 query。
3. 不记录明文 IP。
4. 不记录 Cookie。
5. 不记录 Authorization header。
6. 不记录完整请求头。
7. 不记录用户提交的完整 request body。

允许记录：

1. `targetDomain`
2. `targetScheme`
3. `targetQueryPresent`
4. `targetPathHash`
5. `targetUrlHash`
6. `clientIpHash`

## 后续实现建议

1. 在请求入口生成或透传 `requestId`。
2. 在创建接口增加 `short_link_create_succeeded` 和 `short_link_create_failed`。
3. 在跳转接口增加 `short_link_redirect_succeeded` 和 `short_link_redirect_failed`。
4. 在访问计数更新处增加 `short_link_visit_count_updated` 和 `short_link_visit_count_update_failed`。
5. 所有日志字段使用 allowlist 方式构造，不直接打印请求对象。
